### 💻 AutoForge 技术架构解析 (Architecture & Engineering)

**页面整体视觉基调：** 极简的工业暗黑风（深灰背景），使用流畅的细线、微发光的节点（代表数据流）和等宽字体，呈现类似 IDE 界面的冷静克制感。

#### 1. 核心设计理念 (Core Philosophy)
* **标题：AI 辅助生成与确定性控制的物理隔离**
* **技术简述：** 通用大模型擅长逻辑推演，但存在概率性偏差。AutoForge 的核心架构是通过“沙漏型双层机制”，将 AI 的代码生成能力约束在应用层，而将涉及物理安全的边界校验下沉至底层的确定性沙盒中。
* **视觉动效：** 一个直观的上下分层图。上层是“动态生成的 C# 应用层代码”，下层是“固化的 C++ 防护底座”，中间用一道带锁的半透明屏障（API 层）隔开。

#### 2. 系统管线工作流 (The 4-Stage Pipeline)
* **视觉呈现：** 网页中央是一个从左向右延伸的水平流程图。当用户向下滚动时，流程图的四个节点依次被高亮激活。

* **节点 1：多模态数据输入 (Data Ingestion)**
    * **功能：** 解析工程师上传的 Excel BOM 表、I/O 地址映射表及 MES API 接口文档（如 Swagger/PDF）。
    * **产出：** 自动进行数据清洗，生成系统内部的全局标准变量字典。
* **节点 2：多智能体图网络编排 (Agentic Workflow)**
    * **功能：** 基于 LangGraph 架构。主路由节点分析需求后，将任务分发给平行的专家 Agent（运动控制、视觉、IT 数采）。
    * **校验机制：** 内置静态逻辑检查节点。若发现并行任务间存在时序冲突，流程将在图网络内部反向回退（回溯重写），直至输出无冲突的逻辑。
    * **产出：** 标准化的《机器蓝图协议》(Machine Blueprint JSON)。
* **节点 3：确定性编译与人在回路 (Compilation & HITL)**
    * **功能：** 读取 JSON 蓝图，逆向渲染为可视化工艺连线图（SFC）。**此阶段强制要求现场工程师进行人工复核与放行。**
    * **产出：** 通过代码模板引擎，将 JSON 稳定转化为符合语法的 C# 工程文件及 UI 界面。
* **节点 4：安全底座链接 (Sandbox Linking)**
    * **功能：** 将生成的 C# 业务代码与 AutoForge 闭源核心 DLL 进行动态链接并编译打包，生成最终的可执行程序。

#### 3. 核心护城河：底层物理沙盒剖析 (The Security Sandbox)
* **视觉呈现：** 代码切分对比图。左侧展示带有潜在越界错误的控制代码，右侧展示底层 C++ DLL 是如何拦截该脉冲的内部机制示意图。

* **机制 A：硬互锁防撞引擎 (Hardware Interlock Engine)**
    * **原理描述：** 平台要求预先录入设备极限空间包络面参数。在向底层总线发送实际控制脉冲前，DLL 会进行微秒级 (μs) 的干涉运算。一旦检测到动作将导致干涉，系统会直接在底层切断输出并抛出异常。
* **机制 B：IT/OT 异步隔离 (Asynchronous Isolation)**
    * **原理描述：** 工业控制的主节拍（Cycle Time）极度敏感。我们将所有 MES 网络通讯、日志上报封装在独立的底层异步线程和 SQLite 本地缓存中。即使遇到工厂断网或网络高延迟，核心运动控制也绝不会出现卡顿。

#### 4. 扩展性与生态：硬件抽象层 (HAL & Skills)
* **视觉呈现：** 一个六边形蜂窝结构的拓展图。中心是 AutoForge Core，周围拼接着不同品牌的硬件图标。
* **技术简述：** 平台通过定义统一的“Skill 插件协议”实现软硬件解耦。第三方硬件厂商（相机、伺服）只需按照协议提供驱动接口，即可被 AI 识别并供工程师通过自然语言直接调用，无需修改平台底层核心。
