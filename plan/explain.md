*利用网格卡片对比现状与 AutoForge 介入后的改变：*
* **痛点 1：代码风格碎裂 vs. 100% 绝对统一**
    * *现状：* 不同软件工程师适配非标动作时风格各异，离职交接如同灾难。
    * *AutoForge 赋能：* AI 严格学习贵司的编码规范模板。生成的适配层代码，命名、异常捕获、日志注入 100% 像同一个最严谨的总工写出来的。
* **痛点 2：人为遗漏 vs. 穷举式安全（报警/权限）生成**
    * *现状：* 赶交期时极易漏写边缘气缸的防撞报警或操作员降级权限，导致现场客诉。
    * *AutoForge 赋能：* 基于贵司硬件拓扑，AI 强制进行全量状态机遍历，自动生成所有极限报警逻辑与权限树，消灭“工程师的粗心”。
* **痛点 3：软件人力瓶颈 vs. 释放核心研发算力**
    * *现状：* 资深软件大牛被困在改点位、调 UI 等低级适配中，新机型排期极其困难。
    * *AutoForge 赋能：* 将适配权下放！只需导入新机型 I/O 表和组件清单，初级工程师即可用自然语言生成适配代码，让大牛回归核心算法研发。

### 📡 第三屏 (The Nightmare Killer)：跨越 MES 对接的无底洞
* **大标题：** 连线即上云，告别 100 页的 MES 接口文档。
* **副标题：** 扔进甲方提供的 API 文档，AutoForge 在 3 分钟内为您生成 100% 贴合现有框架的 MES 通信类、数据映射关系与断点续传逻辑。
* **直击痛点的三大特性：**
    * **📄 文档即代码 (Doc-to-Code)：** 大模型深度阅读 Swagger/PDF 文档，自动生成包含鉴权、Token 刷新的全套 HTTP/MQTT 通讯模块，直接并入贵司框架。
    * **🛡️ 异步沙盒，绝不卡机 (IT/OT Decoupling)：** 生成的底层逻辑自动隔离控制线程与网络线程。厂区网络再烂，主轴节拍照样全速跑。
    * **📦 断网本地缓存 (Auto-Resume)：** 自动生成 SQLite 本地转储逻辑。网络中断时数据自动落盘，恢复后静默续传，确保甲方要求的数据“一条不丢”。

### 🔌 第四屏 (How it Works)：非侵入式工作流
* **步骤一：导入资产 (Ingest)：** 将贵司的“标准 UI 控件库”、“硬件驱动”注册到 AutoForge 环境。
* **步骤二：意图拼装 (Assemble)：** 工程师输入新订单差异（如：“新增两个 CCD 飞拍工位，复用现有视觉插件”）。
* **步骤三：生成胶水代码 (Generate)：** 精准生成枯燥的胶水层代码（实例化对象、绑定 ViewModel、注册 I/O 监听）。
* **步骤四：完美编译 (Compile)：** 生成的代码无缝合并入贵司的 `.sln` 工程，一键编译，直接跑机。
